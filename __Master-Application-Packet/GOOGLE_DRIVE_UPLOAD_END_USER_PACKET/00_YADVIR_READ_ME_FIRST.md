# Yadvir Read Me First — Compliance Submission Guide

Start here before opening application folders.

## Working packet rule

- Use each lane's compiled signer-review binder and official forms as the working packet.
- Use each lane's compiled course materials for quick reviewer access to the course content.
- Use each lane's Moodle LMS reference folder only as implementation/reference support.
- Do not use files in `99_NOT_FOR_SUBMISSION_REFERENCE` for submission.

## Do not submit

- Raw source folders
- Build scripts
- Generated fragments
- Old individual module PDFs
- Reference-only exclusions

## Approval status

Government approval is still pending across all lanes. No course should be advertised as approved until the applicable agency approval is issued.

## Final remaining items are human/regulator items only

- Final signatures, initials, and dates
- Final payment/check insertion where required
- Final reviewer access verification where required
- Final legal/privacy signoff where required
- Vanessa evidence attachment verification
- Agency approval / issued number(s)

## Vanessa Evidence Verification Checklist

### 1. Already provided in source — verify and attach

- Resume/CV or professional experience documentation already present in source-backed evidence folders
- RN license / license number `788389` where source-backed
- Instructor / SME source references already used in the packet
- Source-backed instructor evidence indexes already in the master packet

Mark these as: **Provided in source — Yadvir to verify currentness, completeness, and correct attachment before submission.**

### 2. Needs Vanessa confirmation only if outdated/missing

- Confirm RN license is current at time of submission
- Confirm CV/resume is current enough for submission
- Confirm any required attestation/signature fields are complete if the agency requires them

### 3. Final signer/submission verification

- Verify the evidence packet is attached in the correct lane
- Verify attachment readability/completeness
- Verify final signer packet, forms, and checklists all align with the same instructor identity
